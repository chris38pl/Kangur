## Mobile — UX

- [x] **Shopping mode — category list hint:** tap category to open products (`shoppingMode.tapCategoryHint`).
- [x] **Shopping mode — aisle swipe hint:** swipe right = bought, left = not bought (`shoppingMode.itemSwipeHint`). All 10 locales.
- [x] **Shopping leave → list editor:** „Powrót do edycji listy” skips exit sheet; leave always `replace`s to `/list/{id}` (resume-from-Home no longer drops to Home).
- [x] **Navigation principles:** Root / Task / Details; Intent-owned exits (`finishTask`, `leaveShoppingTask`, …). See [navigation-principles.md](../navigation-principles.md).
- [x] **Android Back on Root:** Home double-back exit toast; other tabs → Home.
- [x] **History filters UI:** badge chips matching list-edit filters.
- [x] **Lists history sync:** background sync label (no focus spinner); pull-to-refresh only for user pull.
- [x] **Home AI credits:** compact line under workspace banner + progress.
- [x] **List AI credits hint:** Premium row shows **Kredyty AI** + remaining badge.
- [x] **Premium — Stripe test card helper:** DEV/preview only bottom sheet with `4242…`.
- [x] **Admin — Workspace browser:** Platform → przegląd przestrzeni (list/search/enter/edit/delete).
- [x] **Workspace invite — success sheet:** branded sheet after send/resend (no share/copy-link).
- [x] **Branded feedback sheets:** shared `FeedbackSheet` for success / destructive / info.
- [x] **List item status icons:** check/cross overlay on category emoji (no gray cart column).
- [x] **Meal proposal:** uncheck-hint above ingredients.
- [x] **Shopping finished notification:** list icon matches Home brand bag.
- [x] **Performance (Fale 0–2):** `perf.*` timers; slim DTOs; RQ hydrate; batch AI apply; screenshot quality + AI phase labels.
- [x] **Realtime polling focus gate:** subscribe only while focused; `goRoot` replace for clean unmount.
- [x] **AI apply → scroll to items.**
- [x] **Notification center:** no full-screen spinner on entry.
- [x] **App update soft gate:** `GET /api/v1/app/version` + in-app update sheet.

## Backend / security

- [x] **Invite token hygiene:** no raw token in create API / notifications / push / logs; preview **POST**; accept by `token` XOR `invitationId`; deep-link token shape validation; single-use accept unchanged.
- [x] **AI credits business logic:** reserve at ingest (OpenAI); refund on failure; 15 min hold TTL (`AiCreditHold`); no debit on apply.
- [x] **Tiered rate limits:** AI / invitations (~20/h) / auth / notifications.
- [x] **Admin bootstrap:** `PLATFORM_ADMIN_EMAILS` promote only when Clerk email **verified**.
- [x] **AI integrity:** apply rehydrates from stored proposal; Prompt/Data Separation delimiters.
- [x] **AuthZ hygiene:** push token no cross-user reassign; `email_mismatch` without invitee email leak; workspace PATCH owner/admin only.
- [x] **Hardening:** screenshot MIME + magic-byte sniff; generic Stripe webhook errors; [security.md](../security.md).
- [x] **Free plan AI credits:** monthly default `15` (`AI_FREE_MONTHLY_CREDITS`).
- [x] **Platform workspaces / AI insights APIs** for console.

## Mobile — store / build config

- [x] **Store application ID:** `preview` / `production` → `app.kangur`; DEV → `app.kangur.dev`. Dev icons + `.env.example` aligned.
